CONTEXT
───────
Section 1 produced nonsensical results (368M tok/s, nanosecond latencies)
because GPU work was dispatched asynchronously but never waited on. Memory
measurement failed on macOS. Several minor issues also needed cleanup.

All 5 fixes were applied and verified on the merging branch.


FIX 1: ADD GPU SYNC BARRIERS  [CRITICAL] — DONE
─────────────────────────────────────────────────
File: src/benchmark.rs, benchmark_model() function

Root cause: `let _ = model.forward(input);` dropped the tensor without syncing.
WGPU dispatches GPU kernels async — timer captured only dispatch overhead.

Fix: Added `B::sync(device)` after each model.forward() in both warmup and
benchmark loops. For WGPU it blocks until GPU finishes; for NdArray it's a no-op.


FIX 2: CROSS-PLATFORM MEMORY MEASUREMENT — DONE
─────────────────────────────────────────────────
File: src/benchmark.rs, MemoryStats::current()

Root cause: Read Linux-only /proc/self/statm, failed on macOS.

Fix: Replaced with `sysinfo` crate (already in Cargo.toml). Now uses
get_current_pid() + process.memory() / process.virtual_memory().


FIX 3: HARDCODED WRONG BURN VERSION — DONE
───────────────────────────────────────────
File: src/main.rs — changed "0.14" to "0.20.1".


FIX 4: DUPLICATE model_configs — DONE
──────────────────────────────────────
File: src/main.rs — removed first declaration that was immediately overwritten.


FIX 5: REMOVE UNUSED sys-info DEPENDENCY — DONE
────────────────────────────────────────────────
File: Cargo.toml — removed sys-info = "0.9".


VERIFIED RESULTS (macOS aarch64, Apple Silicon)
───────────────────────────────────────────────
Before fix → After fix:

  NdArray tiny:   1.20µs / 106M tok/s   →   2.14ms / 59,775 tok/s
  WGPU tiny:      347ns / 368M tok/s     →   2.82ms / 45,398 tok/s
  NdArray small:  (not recorded)         →  10.04ms / 12,746 tok/s
  WGPU small:     (not recorded)         →   4.63ms / 27,635 tok/s

Key observations:
  - WGPU beats NdArray on the larger (small) model: 4.63ms vs 10.04ms
  - NdArray beats WGPU on the tiny model: GPU dispatch overhead > compute
  - WGPU tiny (2.82ms) same order of magnitude as PyTorch MPS (1.25ms)
